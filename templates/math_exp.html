Image Processing

Theory

When it comes to digital image-processing, there's a lot of math involced behind the scenes. An digital image is made of pixels, square like units that store values. These pixels are presented in an order array, like on a graph, thereby creating an image.

BUT, the pixel can further be broken down into several categories - black and white, colored etc. These subcategories also have individual subdivsions, for example, the color can further be broken down into Hue, Saturation and Value. The task of displaying these images can be done by most modern applications but if one has to process the image first:

With 1920 pixels in a row, 1080 pixels in a column,
roughly 2 million pixels overall.

All of the above mentioned subdivions of each of these 2 million pixels has to be modified about 25 times a second, thereby leaving us with 50 million actions every second.

The following matrix equation takes care of all of the above mentioned tasks.

Equation

Following are two tables for reference of variable names in the equation.

Matrix - I

Variable Meaning
X-coordinate Represents the initial x-coordinate of the pixel as an integer
Y-coordinate Represents the initial y-coordinate of the pixel as an integer
B/W Represents a boolean value (True or False) as to whether or not the image is in Grayscale.
Hue Represents the initial Hue of the pixel as an integer
Saturation Represents the initial saturation of the pixel as an integer
Value Represents the initial value of the pixel as an integer

Matrix -II


Variable Meaning
New_x Represents the new x-coordinate of the pixel as an integer
New_y Represents the new y-coordinate of the pixel as an integer
C.space Represents a boolean value (True or False) as to whether or not the image has been converted to Grayscale or a different color-space.
New.H Represents the new Hue of the pixel as an integer
New.S Represents the new saturation of the pixel as an integer
New.V Represents the new value of the pixel as an integer
